＊내장객체와 영역

웹컨테이너는 jsp페이지에서 상황에 따라 필수적으로 9개의 객체를
객체의 생성없이 바로 사용할수있도록 제공 => 이들 객체들을 jsp의 내장객체

＊내장객체 개요:
jsp페이지 내에서 제공하는 특수한 레퍼런스 타입의 변수로 사용하고자 하는 변수와 메소드 접근

request	: 웹브라우저의 요청정보를 저장하고있는 객체
response	: 웹브라우저의 요청에 대한 응답정보를 저장
out	: jsp페이지 출력할 내용을 가지고 있는 출력 스트림 객체
session	: 하나의 웹브라우저내에서 정보를 유지하기 위한 세션정보를 저장하고있는 객체
---- 위에 4개는 자주 쓰이는것
application
pageContext
page
config
exception	:예외가 발생한 경우 사용

request,session,application,pageContext 속성을 저장하고 있을수있는 메소드인 setAttribute()메소드와 getAtttribute()메소드 제공


 - request: 웹브라우저에서 jsp페이지로 전달되는 정보의 모임 / 웹 브라우저의 요청 정보를 저장

 - response: 웹브라우저로 응답할 응답정보를 가짐/ 응답정보와 관련하여 주로 헤더정보입력, 리다이렉트 하기등의 기능을제공 / 웹 브라우저의 요청에 대한 응답 정보를 저장
 
※헤더: 쓰는이유는 읽는속도를 빠르게 하기 위해서

4byte:
헤더			데이터 	: 하나의 패킷.데이터
2byte		2byte
데이터에		실제데이터
대한 정보

받는쪽에서는 헤더를 먼저 읽음. 데이터가 많아지면 헤더만 읽으면 빠름

 - out: jsp페이지가 웹브라우저에 보내는 모든 정보는 out 객체로 통해서 전송/주로 많이 사용되는건 println() / jsp페이지 출력할 내용을 가지고 있는 출력 스트림

 - session: 요청한 웹 브라우저에 관한 정보를 저장하고 관리하는 내장객체. 로그인할때 많이쓰임 / 하나의 웹 브라우저의 정보를 유지하기 위한 세션 정보를 저장
 	session객체는 웹브라우저(클라이언트) 당 1개가 할당. 따라서 주로 회원관리 시스템에서 사용자 인증에 관련된 작업을 수행할때사용
 	invalidate(): 세션무효화시 사용

 - application: 웹 어플리케이션(톰캣같은서버)의 설정정보를 갖는 context와 관련이 있는 객체로 웹애플리케이션과 연관/ 서버의 설정정보 및 장원에 대한 정보 / 웹 어플리케이션 context의 정보를 저장
웹 애플리케이션 당 1개의 객체가 생성/ 웹사이트 방문자 기록을 카운트할때 사용

 - config: 서블릿이 초기화되는 동안 참조해야할정보를 전달해주는 역할/컨테이너당 1개 / jsp페이지에 대한 설정정보를 저장

 - page: jsp 페이지 그 자체를 나타내는 객체. this 키워드로 자기 자신을 참조 할수 있음/ 현재 거의 사용되지않는 내장객체 / jsp페이지를 구현한 자바 클래스 객체
 
 - pageContext : jsp페이지에 대한 정보를 저장

 - exception: jsp페이지에서 예외가 발생했을때 예외처리 할 페이지에 전달되는 객체 / jsp페이지에서 예외가 발생한 경우에 사용

웹어플리케이션은 page,request, session, application이라는 4개의 영역을 가지고 있음
내장 객체의 영역은 객체의 유효기간이라고도 불리며, 객체 공유범위

page영역: 객체를 하나의 페이지 내에서만 공유
request영역: 한번의 웹 브라우저의 요청에 대해 같은 요청을 공유하는 페이지가 대응
session영역: 같은 웹브라우저 내에서는 요청되는 페이지들은 같은 객체를 공유
